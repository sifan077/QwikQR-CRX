# QwikQR 代码规范

## 代码风格

### JavaScript
- 使用 2 空格缩进
- 使用单引号而非双引号
- 每个语句末尾添加分号
- 使用 ES6+ 特性（const、let、箭头函数等）
- 将常量定义在文件顶部，使用大写字母和下划线命名

### HTML
- 使用 2 空格缩进
- 正确设置 `lang` 属性（中文内容使用 `lang="zh-CN"`）
- 包含 `viewport` meta 标签以支持响应式设计

### CSS
- 使用 2 空格缩进
- 遵循属性按字母顺序排列的原则
- 使用有意义的类名

## 命名约定

### 变量和函数
- 常量使用 `UPPER_SNAKE_CASE`
- 变量使用 `camelCase`
- 函数使用 `camelCase`
- 私有变量/函数前缀使用下划线（如有需要）

### 文件命名
- 使用 kebab-case 命名文件（例如：`service-worker.js`）
- HTML/CSS 文件与对应的 JavaScript 文件同名

## 代码组织

### 文件结构
```
- 常量定义
- 工具函数
- 主要功能函数
- 事件监听器
```

### 函数
- 每个函数应该只做一件事
- 函数长度应该控制在 50 行以内
- 复杂逻辑应该拆分成多个小函数

## 最佳实践

### 错误处理
- 使用 try-catch 块处理可能抛出异常的代码
- 提供友好的错误消息给用户
- 在 console 中记录详细的错误信息

### 性能
- 使用防抖（debounce）处理高频事件
- 及时清理不再使用的资源（如 blob URLs）
- 避免不必要的 DOM 操作

### 用户体验
- 提供清晰的状态反馈（加载中、成功、失败）
- 使用通知而非 alert() 弹窗
- 适当的过渡动画和视觉反馈
